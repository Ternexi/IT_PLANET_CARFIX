# CarFix Application

**CarFix Application** — это мобильное приложение для Android, предназначенное для автоматизации учета кузовного ремонта автомобилей. Оно позволяет создавать заказы на ремонт, отслеживать их статус и хранить всю историю операций на удаленном сервере (бэкенде).

---

## 1. Для чего это приложение?

Приложение разработано для упрощения взаимодействия между мастерами кузовного цеха и системой учета. Основные возможности:
*   **Автоматизация учета**: Избавление от бумажной волокиты при регистрации новых заказов.
*   **Создание заказов**: Мастер может быстро создать новый заказ, указав данные автомобиля и перечень работ.
*   **Хранение истории**: Все данные сохраняются на бэкенде, что позволяет найти историю ремонтов любого автомобиля по его государственному номеру.
*   **Прозрачность**: Быстрый доступ к деталям заказа (стоимость, время выполнения, список запчастей и работ).

## 2. Как собрать и протестировать приложение

Для того чтобы запустить приложение на компьютере для тестирования, выполните следующие шаги:

### Системные требования
*   Установленная **Android Studio** (рекомендуется версия Hedgehog или новее).
*   **JDK 11** или выше.
*   Доступ к интернету (для загрузки библиотек и связи с сервером).

### Клонирование и сборка проекта

1.  **Клонируйте репозиторий**:
    *   **Через терминал**: Введите команду:
        ```bash
        git clone https://github.com/Ternexi/IT_PLANET_CARFIX.git
        ```
    *   **Через Android Studio**: На стартовом экране выберите **"Get from VCS"** (или **"Clone Repository"**), вставьте ссылку `https://github.com/Ternexi/IT_PLANET_CARFIX.git` и нажмите **Clone**.

2.  Дождитесь завершения **Gradle Sync** (процесс загрузки необходимых компонентов, индикатор внизу экрана).

3.  **Сборка APK-файла**:
    *   В верхнем меню выберите: `Build` -> `Build Bundle(s) / APK(s)` -> `Build APK(s)`.
    *   После завершения сборки появится уведомление со ссылкой на файл (путь: `app/build/outputs/apk/debug/app-debug.apk`).

### Запуск для тестирования
*   **На эмуляторе**: В Android Studio выберите `Device Manager`, создайте виртуальное устройство и нажмите кнопку "Run" (зеленый треугольник).
*   **На реальном устройстве**: Подключите телефон по USB, включите "Отладку по USB" в настройках разработчика и нажмите "Run".

## 3. Используемые технологии

Приложение написано с использованием современных инструментов разработки:
*   **Kotlin** — основной язык программирования.
*   **Retrofit 2 & OkHttp** — работа с API сервера.
*   **Coroutines** — асинхронное выполнение запросов.
*   **ViewBinding** — связь кода с элементами интерфейса.
*   **RecyclerView** — отображение списков заказов.
*   **Gson** — обработка JSON-данных.
*   **Material Design** — графический интерфейс.

## 4. Структура проекта

### Схематичный вид
```text
CarFixApplication/
├── app/
│   ├── src/
│   │   ├── main/
│   │   │   ├── java/com/example/carfixapplication/
│   │   │   │   ├── api/            # Сетевой слой (интерцепторы)
│   │   │   │   ├── App.kt          # Инициализация приложения
│   │   │   │   ├── MainActivity.kt # Главный экран
│   │   │   │   ├── LoginActivity.kt
│   │   │   │   ├── RegisterActivity.kt
│   │   │   │   ├── SearchCarActivity.kt
│   │   │   │   └── HistorySearchActivity.kt
│   │   │   └── res/
│   │   │       ├── layout/         # XML разметка экранов
│   │   │       └── values/         # Ресурсы (строки, цвета, темы)
│   ├── build.gradle.kts            # Конфигурация модуля app
└── build.gradle.kts                # Корневой файл сборки
```

### Подробное описание
*   **`app/src/main/java/com/example/carfixapplication/`** — Основной код приложения:
    *   `api/` — содержит настройки подключения к серверу и AuthInterceptor для авторизации.
    *   `MainActivity.kt` — центральный экран управления.
    *   `LoginActivity.kt` / `RegisterActivity.kt` — работа с профилем пользователя и доступ в систему.
    *   `SearchCarActivity.kt` — интерфейс создания нового заказа.
    *   `HistorySearchActivity.kt` — экран поиска истории обслуживания.
*   **`app/src/main/res/layout/`** — Содержит XML-файлы интерфейса. Например, `activity_main.xml` определяет, как выглядит главный экран.
*   **`app/build.gradle.kts`** — Здесь прописаны все зависимости (Retrofit, Coroutines и др.), которые нужны для работы приложения.
